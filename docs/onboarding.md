# Протокол онбординга

> Читается один раз — при первом запуске нового проекта, скопированного из
> шаблона `_b24-single-php`, пока не пройден `.onboarding-done`.
> После онбординга этот файл больше не нужен для работы — можно оставить
> как есть (не мешает) или удалить.

Задавай вопросы **строго по одному**. Если ответ очевиден из контекста — уточни, не пропускай.

### Вопрос 1 — Название и суть
*"Как называется проект и что он делает? Одной-двумя фразами."*

*(Пример: «Пульт сделок — дашборд аналитики CRM для менеджеров»)*

### Вопрос 2 — Портал Б24
*"На каком портале Битрикс24 работает клиент?"*

*(Пример: company.bitrix24.ru)*

### Вопрос 3 — Плейсменты
*"Где откроется приложение в Битрикс24?"*

Варианты:
- **LEFT_MENU** — пункт в левом меню (дефолт, уже настроен в `www/api/bind.php`)
- **CRM_DEAL_DETAIL_TAB** / **CRM_CONTACT_DETAIL_TAB** — вкладка в карточке сделки / контакта
- **CRM_DETAIL_ACTIVITY** — виджет активности в карточке CRM
- **Несколько** — укажи список

### Вопрос 4 — Нужен ли cron?
*"Нужна ли фоновая обработка данных (cron-задача)?"*

- **Нет** → удаляю `www/bin/process.php` из проекта
- **Да** → оставляю, заполняю `runJob()` по ходу работы

### После всех ответов:
1. Заполни раздел «О ПРОЕКТЕ» в `CLAUDE.md` — замени все `{{...}}` реальными значениями
2. Обнови `docs/project-brief.md` (таблицы «Общее» и «Технические параметры»)
3. Обнови `docs/state.md` — скорректируй «Следующие шаги» под конкретный проект
4. Если cron не нужен — удали `www/bin/process.php`
5. Создай пустой файл `.onboarding-done` в корне проекта
6. Снимок в git: `bash scripts/snapshot.sh "онбординг: <название проекта>"`
7. Скажи, что настроено и что нужно дальше (обычно: раскатить `www/` → открыть `init.php` → создать local-app в Б24)
